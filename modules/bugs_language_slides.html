The BUGS language

The BUGS language is described in detail in the relevant section of the BUGS manual.

Stochastic declarations:

  • x ~ dgamma(shape, scale) (x follows a gamma distribution with parameters shape and scale)

Deterministic declarations

  • y <- 2 * x

For loops

  • for(i in 1:10) {x[i] <- exp(y[i])}

A basic example

We'll use the pump model example from the original BUGS software. The data set describes the failure rates of some pumps.

In this model, there are pumps. The number of failures, , for pump is assumed to follow a Poisson distribution, with mean where is the length of operation of the pump in thousands of hours and is the failure rate. The model then borrows strength across pumps with a prior that ties the random effects together using a common distribution, . and are the hyperparameters of this random effects distribution.

Specifying the BUGS code for a model

Here we specify the pump code directly in R. We can walk through some of details via the comments in the BUGS code.

library(nimble)
pumpCode <- nimbleCode({ 
  for (i in 1:N){

      # likelihood (data model)
      x[i] ~ dpois(lambda[i])

      # latent process (random effects)
      # linear predictor      
      lambda[i] <- theta[i]*t[i]
      # random effects distribution
      theta[i] ~ dgamma(alpha,beta)
  }
  # priors on hyperparameters
  alpha ~ dexp(1.0)
  beta ~ dgamma(0.1,1.0)
})
